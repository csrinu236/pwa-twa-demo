Caching json responses VS caching file respones(like about.js, files.html, etc) are NOT same.

PWA install button gets enabled only after website is ready enough to be installed as a PWA (i.e. when beforeinstallprompt event triggers, corresponding callback functions gets executed and in the function, we set install button disabled = false). If the PWA is already installed then beforeinstallprompt event will never fires.

(Fetching & dynamically/on-the-fly caching of JSON respones) Caching fetch json responses VS caching file respones(like about.js, files.html, etc) are NOT same. This fetch button is to show how dynamic caching or on-the-fly caching of json responses works and serving caches response for subsequent requests.

This button is to show how manually we can cache files responses on demand by clicking a button. caches.add('/manifest.json') actually sends fetch request to get the file and cache it.
